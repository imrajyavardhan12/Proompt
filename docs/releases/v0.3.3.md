## What's Changed

### Unsigned macOS Accessibility onboarding

- Settings now gives clearer step-by-step guidance for enabling selected-text Quick Enhance on unsigned macOS builds.
- The Accessibility reset command is copyable directly from Settings.
- Settings explains that unsigned builds may need Accessibility reset/re-grant after replacing or rebuilding the app.
- README now points users to Selected-text diagnostics for recovery.

### Release workflow hygiene

- Release publishing now creates a draft first, uploads all assets, and publishes only after build/upload jobs succeed.
- Curated release notes are now supported via `docs/releases/vX.Y.Z.md` so releases keep the structured v0.3.x style.
- macOS signing/notarization support is prepared for the future, but Developer ID signing is optional and deferred.
- Unsigned DMG releases remain the expected path while there is no Apple Developer account.
- The macOS release job now requires explicit `PROOMPT_ENABLE_MACOS_SIGNING=true` before any Apple signing secrets are passed to Tauri, preventing stale/invalid Apple secrets from breaking unsigned releases.

### QA runbook

- Added `docs/unsigned-macos-qa.md` with a repeatable smoke matrix for unsigned Quick Enhance selected-text behavior.
- Release checklist now calls out that runbook when selected-text behavior changes.

### macOS Accessibility and diagnostics

Selected-text capture/replacement on macOS requires Accessibility permission.

Proompt writes a local troubleshooting file at:

`~/Library/Application Support/proompt/selection-diagnostics.json`

This file contains only metadata such as capture status, error codes, active app/window metadata, and character counts. It does **not** contain selected text or prompt content. The file stays on your device and can be deleted anytime.

If an unsigned build appears enabled in Accessibility settings but stops capturing selections after an update/rebuild, reset permission and grant it again:

```bash
tccutil reset Accessibility com.proompt.desktop
open /Applications/Proompt.app
```

### Desktop release note

- Desktop DMG is macOS Apple Silicon only for now.
- Desktop app is unsigned and not notarized.
- Unsigned releases are expected while Developer ID signing is deferred.
- `v0.3.2` was not published because the release job exposed a stale Apple signing secret to Tauri; `v0.3.3` contains the same user-facing improvements plus the workflow fix.
- If macOS says the app is damaged, run:

```bash
xattr -dr com.apple.quarantine /Applications/Proompt.app
open /Applications/Proompt.app
```
